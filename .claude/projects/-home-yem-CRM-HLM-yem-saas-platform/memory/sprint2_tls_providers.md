Sprint 2 (2026-03-14): Real Messaging Providers + HTTPS implementation.

**Why:** Production readiness — JWT tokens and magic-link tokens travel in headers/URLs and must be encrypted in transit.

**How to apply:** Use these patterns for any future TLS or messaging-related work.

## TLS
- `SSL_ENABLED=true` activates embedded Tomcat TLS (Mode A, dev/staging). Port 8443.
- `FORWARD_HEADERS_STRATEGY=FRAMEWORK` is set in `application-production.yml` for Mode B (Nginx).
- `TlsRedirectConfig` bean active when `server.ssl.enabled=true` — adds HTTP connector that redirects to HTTPS.
- HSTS in `SecurityConfig` is now conditional: `if (sslEnabled)` — never emitted over plain HTTP.
- Test keystore: `src/test/resources/ssl/hlm-dev.p12` generated by Maven exec plugin during `generate-test-resources`.
- `TlsConfigIT` extends `IntegrationTestBase` (Testcontainers PostgreSQL) + uses `RANDOM_PORT` web env.

## Twilio SMS
- `TwilioSmsSender` activated by `@ConditionalOnProperty(name="app.sms.account-sid", matchIfMissing=false)`.
- `Twilio.init()` called lazily in constructor (not at class load).
- Unit test uses `Mockito.mockStatic(Twilio.class)` and `Mockito.mockStatic(Message.class)`.

## GreenMail SMTP tests
- `SmtpEmailSenderIT` and `PortalMagicLinkEmailIT` extend `IntegrationTestBase` (Testcontainers PostgreSQL needed).
- GreenMail on port 3025 via `ServerSetupTest.SMTP`.
- Override `spring.mail.properties.mail.smtp.auth=false` and `starttls.enable=false` in `@TestPropertySource`.
- GreenMail must be started `@BeforeAll` (static) and stopped `@AfterAll`.

## Portal magic-link URL
- URL changed from `/portal/login?token=` to `/portal/verify?token=` in Sprint 2.
- `PortalAuthIT` assertion updated accordingly.
- Frontend: `/portal/verify` route added pointing to `PortalLoginComponent`.

## Migration 028
- Drops v1 payment tables: `payment → payment_call → payment_tranche → payment_schedule` (FK order).
- Each changeset uses `preConditions: onFail: MARK_RAN` and `tableExists` check.
