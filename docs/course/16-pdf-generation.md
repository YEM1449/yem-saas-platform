# Module 16 — PDF Generation

## Learning Objectives

- Identify the two PDF documents generated by the platform
- Explain the openhtmltopdf + Thymeleaf rendering pipeline

## Rendering Pipeline

```
1. Service builds a DocumentModel (POJO with all template variables)
2. Thymeleaf processes the HTML template → HTML string
3. openhtmltopdf renders HTML string → PDF bytes
4. Controller returns bytes as application/pdf response
```

### Thymeleaf Template

```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head><title>Reservation</title></head>
<body>
  <h1>Bon de Réservation</h1>
  <p>Référence: <span th:text="${deposit.reference}"></span></p>
  <p>Acheteur: <span th:text="${contact.fullName}"></span></p>
  <p>Montant: <span th:text="${deposit.amount}"></span> MAD</p>
</body>
</html>
```

### openhtmltopdf Rendering

```java
try (ByteArrayOutputStream os = new ByteArrayOutputStream()) {
    PdfRendererBuilder builder = new PdfRendererBuilder();
    builder.withHtmlContent(htmlContent, "classpath:/templates/");
    builder.toStream(os);
    builder.run();
    return os.toByteArray();
}
```

## The Named Entity Pitfall

**openhtmltopdf is an XML parser.** It uses XHTML parsing rules, which do NOT support HTML4 named entities like `&nbsp;`, `&mdash;`, `&euro;`, etc.

If a Thymeleaf template contains:
```html
<p>Total&nbsp;MAD</p>
```

The PDF generation throws:
```
com.openhtmltopdf.util.XRRuntimeException: Element [html] not found, unable to render
```

**Fix:** Use numeric character references instead:

| Named entity | Numeric ref | Character |
|-------------|-------------|-----------|
| `&nbsp;` | `&#160;` | Non-breaking space |
| `&mdash;` | `&#8212;` | Em dash — |
| `&euro;` | `&#8364;` | Euro sign € |
| `&laquo;` | `&#171;` | « |
| `&raquo;` | `&#187;` | » |
